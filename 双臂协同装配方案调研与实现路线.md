# 双臂协同插拔/装配任务：方案调研与实现路线

> 整理日期：2026-07-03
> 硬件条件：UR5 + UR10（均为 CB 系列，**无内置力/力矩传感器**）
> 目标任务：插座拔插、螺丝对准放入螺母（一臂固定工件，另一臂操作）
> 已确定路线：先做方案一（视觉 + 导纳力控），后续叠加模仿学习

---

## 一、文献调研：三条主流技术路线（2022–2025）

任务定位：**双臂非对称协作的富接触（contact-rich）插入任务**。近两年学术主流已转向"模仿学习 + 柔顺控制"；工业界最稳妥的仍是"视觉定位 + 力/阻抗控制 + 主从协调"。

### 路线一：传统方案 —— 视觉定位 + 力控搜孔 + 主从协调（无需学习）

固定臂（master）刚性夹持工件保持不动，操作臂（slave）用**阻抗/导纳控制**做柔顺插入，配合螺旋搜孔（spiral search）消除定位误差。

- 主从协调拧螺母可追溯到 Su et al. 2009（见 [dual-arm learning 综述, 2026](https://www.sciencedirect.com/science/article/pii/S1367578826000064)）
- [双臂互相视觉跟踪定位完成螺丝-螺母装配（2023, PMC）](https://ncbi.nlm.nih.gov/pmc/articles/PMC10450283)：两臂各带相机互相跟踪定位，传送带场景下完成螺丝螺母装配
- [大型物体多 peg-in-hole 双臂协作框架（RCIM 2025）](https://dl.acm.org/doi/10.1016/j.rcim.2025.102991)
- [双臂对称任务的自适应协调阻抗控制（2025）](https://www.sciencedirect.com/science/article/abs/pii/S0921889025002076)：接触力分解为内力/外力分别设计阻抗自适应策略

**优点**：不需要采数据，可靠、可解释、精度高。**缺点**：换插头/螺丝规格要重新标定调参，泛化差。

### 路线二：强化学习 + 变阻抗控制

- [Learning to Centralize Dual-Arm Assembly（Frontiers in Robotics and AI, 2022）](https://www.frontiersin.org/journals/robotics-and-ai/articles/10.3389/frobt.2022.830007/full)：与本任务形态最接近的 RL 工作。两台 Franka，一臂持轴一臂持孔，SAC 学中心化策略输出两臂变阻抗控制目标，间隙 0.5–2mm，仿真训练零样本迁移真机，笛卡尔阻抗动作空间真机效果最好
- [DA-VIL（2024）](https://arxiv.org/html/2410.19712v1)：RL（PPO）+ 变阻抗 QP 控制器，面向双臂共同搬运，可参考其"策略输出刚度参数"架构
- [FILIC（2025）](https://arxiv.org/pdf/2509.17053)：力引导模仿学习 + 阻抗力矩控制双环框架

**优点**：能学到搜孔等接触策略，对定位误差鲁棒。**缺点**：需要仿真环境 + sim-to-real，工程量大。
**注意**：这些工作依赖 Franka 的力矩控制接口，CB 系列 UR 不开放力矩控制，无法直接复现。

### 路线三：模仿学习（2023 年以来的主流）—— 遥操作采数据 + ACT / Diffusion Policy

斯坦福 [ALOHA / Mobile ALOHA](https://arxiv.org/html/2401.02117v1) 带火的路线：低成本双臂遥操作采几十到几百条演示，训练 ACT（Transformer 动作分块）或 Diffusion Policy 直接输出双臂动作。

- 原版 ALOHA 电池插入 78%；精密插入（USB、peg-in-hole）约 200 条演示可到 70%+（[act-plus-plus 仓库](https://github.com/MarkFzp/act-plus-plus)、[双臂模仿学习算法对比, 2024](https://arxiv.org/html/2408.06536v1)——结论：ACT 和 Diffusion Policy 是双臂任务最优选择）
- [Trossen 双臂 ACT peg 插入实验（2024）](https://www.trossenrobotics.com/post/exploring-precision-with-peg-insertion-using-bimanual-robots-an-experiment-with-the-act-model)：一臂持孔一臂持轴，80% 成功率
- **[Comp-ACT（IROS 2024, OMRON SINIC X）](https://omron-sinicx.github.io/CompACT/)**（[arXiv:2406.14990](https://arxiv.org/abs/2406.14990)，[代码开源](https://github.com/omron-sinicx/CompACT)）：**最推荐精读，与本任务几乎一模一样**。ACT 基础上让策略同时输出轨迹和**时变刚度参数**，喂给柔顺控制器；带力反馈 VR 遥操作采数据。双臂 UR5e，圆柱 peg-in-hole 仅 20 条演示成功率 100%，方形 30 条 70%。消融实验证明**力/力矩输入对插入任务至关重要**（有力觉 100% vs 无力觉 40%）
- 更前沿的 VLA 大模型路线（π0、RDT-1B 等）对毫米级插入目前精度不如专用方案

**优点**：不用建模不用仿真，换任务只需重采几十条演示。**缺点**：需要遥操作设备；纯视觉对毫米级插入精度不够（需加力觉）。

持续跟踪资源：[awesome-bimanual-manipulation](https://github.com/Skylark0924/awesome-bimanual-manipulation)（按学习/建模/规划控制/遥操作分类的双臂论文合集）

## 二、针对 UR5 + UR10（CB 系列）的方案选择

**结论：以路线一为主体落地，之后在其力控底层上叠加 Comp-ACT 类模仿学习。**

### 关键硬件约束

1. **CB 系列没有内置力/力矩传感器**，关节电流估计末端力精度仅 ±10–20 N，做毫米级插入完全不够。而力觉对插入任务几乎是刚需（Comp-ACT 消融：有力觉 100% vs 无力觉 40%）。→ **第一件事：给操作臂加装腕部六维力传感器**（Robotiq FT 300-S、ATI Axia80、OnRobot HEX 都是 UR 生态成熟选择）。
2. **UR 不开放关节力矩控制**，只能通过 URScript / RTDE 做位置/速度流控制（CB3 为 125 Hz）。所以"阻抗控制"要实现成**导纳控制**：读外置力传感器 → 导纳模型算位置修正 → `servoj`/`speedl` 流给机械臂。UR 上有大量现成实现（`ur_rtde` 库 + Python 即可）。这也排除了路线二的直接复现。

### 双臂分工

- **UR10 = 固定臂**：负载大（10 kg）、刚性好，夹持插座/螺母做纯位置保持，任务中不动，不需要力传感器。本质上是个"可编程夹具"。
- **UR5 = 操作臂**：装六维力传感器 + 电动夹爪（如 Robotiq 2F-85），负责"视觉粗定位 → 接触 → 螺旋搜孔 → 柔顺插入"。

任务非对称、两臂无需实时协调运动，系统复杂度接近单臂插孔问题。

### 方案一实施路径

1. **硬件补齐**：UR5 腕部六维力传感器 + 两个夹爪 + 眼在手（eye-in-hand）或全局相机
2. **标定**：双臂基座间外参标定（一次性）+ 手眼标定
3. **流程**：相机定位插座孔/螺母位姿（误差 ±2–3 mm 内即可）→ UR5 移到孔上方 → 导纳控制下压接触 → 力引导螺旋/李萨如搜孔 → 落入检测（Z 向突降 + 侧向力特征）→ 柔顺插入；拔出反向同理
4. 换插头/螺丝规格只需改夹持点和搜孔参数

**注意**：若螺丝螺母后续要做"拧入"而不只是"放入"，涉及螺纹啮合的力位混合控制，难度上一个台阶；只是对准放入就是标准 peg-in-hole。

## 三、后续升级到学习方案：需要补什么

方案一的基础设施（力传感器、导纳控制、标定、夹具）**全部复用**。

### 模仿学习（建议的升级路径）

#### 需补硬件

| 项目 | 选择 | 说明 |
|------|------|------|
| 遥操作设备（三选一） | SpaceMouse（千元级） | 6DOF 控末端位姿，小范围精细任务够用，上手最快 |
| | GELLO 主从臂（几千元，3D 打印+舵机） | 官方支持 UR，演示效率和质量最高，**推荐** |
| | VR 手柄（Quest 3） | Comp-ACT 同款，可做力反馈提示，姿态漂移需处理 |
| 相机 | 2–3 个 RGB（RealSense D435 或高帧率 USB 相机） | UR5 腕部 1 个 + 全局 1–2 个；深度非必须 |
| 训练机 | RTX 3090/4090 一张 | ACT 几十条演示训练数小时 |

固定臂不动，只需遥操作 UR5 一条臂，不用买两套。

#### 需做的工作

1. **遥操作管线**：以方案一的导纳控制器为底层，遥操作指令作为期望末端位姿输入——演示时接触力天然受限，安全且数据自带柔顺行为（复用价值最大的一步）
2. **数据采集与同步**：10–30 Hz 记录图像 + TCP 位姿 + 六维力 + 夹爪状态 + 动作指令，存 HDF5/LeRobot 格式。建议直接用 **LeRobot**（HuggingFace）搭采集和训练（ACT/DP 现成实现，自定义机器人只需写接口类），或直接用 Comp-ACT 开源代码（本就是 UR + 力觉）
3. **采演示**：每任务 30–50 条起步（Comp-ACT 20–30 条即可），插座/螺母位置要做随机化，否则学不到纠偏能力
4. **部署**：策略输出末端位姿（Comp-ACT 还输出时变刚度）→ 导纳控制层 → `servoj` 执行。CB3 的 125 Hz 够用：ACT 按动作块（chunk）输出，策略 ~10 Hz 推理一次即可
5. **纪律要求**：从方案一阶段就固定夹爪、相机安装位置和标定，中途变动会作废已采数据

#### 采购清单汇总

**一个 SpaceMouse 或一套 GELLO、1–2 个相机（补到 2–3 视角）、一张 24G 显存 GPU**，其余全是软件工作。

### 强化学习（可选，优先级放低）

CB 系列无力矩控制，Franka 上的力矩级 RL 无法直接复现。现实的两种玩法：

- **仿真训练搜孔策略 + 导纳动作空间**：MuJoCo/Isaac 建 UR5 + 插孔模型，动作空间用"末端位姿增量 + 导纳参数"（与真机接口一致），sim-to-real 鸿沟最小；工程量在建仿真和接触参数辨识
- **在模仿学习策略上做真机微调**（residual RL / RLPD 类）：少量真机交互提升成功率，不需要仿真，需做好安全约束（力阈值急停方案一已有）

建议：模仿学习收益/成本比更高；RL 等模仿学习跑通、发现成功率瓶颈（如特定误差方向总失败）时再作为微调手段引入，不必提前买硬件。

## 四、难度升级：斜孔的圆柱插入（以一定角度插入）

**核心思路：斜孔插入 = 把方案一里所有"默认竖直向下"的逻辑改写到"孔轴坐标系"（task frame）里执行。** 控制算法不变，变的只是参考系。

### 孔轴坐标系 {H}

定义 `{H}`：原点在孔口，Z 轴沿孔轴指向孔内。然后：

- **接近**：末端沿 `{H}` 的 -Z 方向接近，圆柱轴线对齐孔轴
- **导纳控制**：F/T 读数变换到 `{H}` 下再跑导纳模型——插入力沿 `{H}` Z 轴，柔顺放在 `{H}` XY 平面（垂直于孔轴）
- **螺旋搜孔**：螺旋轨迹画在 `{H}` 的 XY 平面上
- **落入检测**：沿 `{H}` Z 向位置突进 + XY 向力特征

代码上即在原管线加一层坐标变换：`F/T 读数 → R_H^T · F`，导纳输出修正量 `→ R_H · Δx` 叠加到基座系目标位姿。

### 孔轴方向的获取（按精度排序）

1. **最优：孔在 UR10 夹持的工件上**——UR10 夹持位姿 + 双臂标定 + 工件 CAD 直接算出孔轴，不需要感知（精度可到 1° 以内）。甚至可以**让 UR10 把工件转到对 UR5 最友好的角度**，"斜孔"直接变回"正孔"。任务允许时优先这么做
2. **孔在固定环境 + 深度相机**：点云拟合孔周围端面法向，或对孔做圆柱拟合
3. **只有 RGB**：孔口圆成像为椭圆，长短轴比给倾角、长轴方向给倾斜方位，配合已知孔径解出 5 自由度位姿（精度 2–5°，够粗对准，剩下靠力控纠偏）

### 三个新增工程要点

1. **力传感器重力补偿（最容易踩的坑）**：末端任意倾斜后，工具+夹爪+圆柱的重力在六通道上的投影随姿态变化，不补偿会被导纳当成虚假接触力。做法：**负载辨识**——UR5 摆 20–30 个不同姿态记录 F/T 读数，最小二乘解出负载质量、质心和传感器偏置，运行时按当前姿态实时扣除
2. **姿态柔顺（角度误差纠正）**：把导纳模型**绕 `{H}` X/Y 轴的转动刚度调低**——圆柱斜卡孔口时接触力矩让末端被动"顺"进孔轴方向，即软件实现的 RCC（远程柔顺中心）。可进一步主动纠偏：检测到持续横向力矩时朝力矩方向微调姿态。柔顺中心设在圆柱尖端而非法兰处，效果好很多
3. **可达性与奇异位形检查**：斜插要求 UR5 腕部特定倾斜姿态，UR 腕部奇异附近 `servoj` 速度会爆。落地前在仿真（RoboDK/MoveIt）里扫一遍目标角度范围内的位形，必要时调 UR5 基座朝向或让 UR10 换持握角度规避

### 对学习路线的影响：几乎免费

ACT/Comp-ACT 学的是末端 6D 位姿轨迹，无"竖直"假设——只需遥操作采斜插演示。想泛化到不同倾角：采数据时让 UR10 把工件摆到不同角度（如 0°–45° 随机），每个角度采若干条。演示底层仍是做好重力补偿的导纳控制，数据天然带柔顺行为。

**注意**：重力补偿、可达性检查无论走哪条路线都要做；搜孔/纠偏策略则可被学习策略取代。

### 实施顺序

负载辨识和重力补偿 → 方案一改成任务系参数化（"插入轴"从常量变输入）→ UR10 摆固定角度逐级验证（15°、30°、45°）→ 再考虑感知估计孔轴或学习泛化。
