## 1. 需求描述（用户原话大意）

改造任务流程为：**UR10 视觉识别桌上工件 → 抓取 → 摆到"UR5 最容易插入"的优化位姿 → UR5 执行插入**。替代目前的"人工放入/固定料位盲抓 + 人工指定呈现位姿"。

## 2. 结论：可行，方案一基础上增量改造，M3 后约 3–5 周

拆为三个独立模块：

| 模块 | 内容 | 难度 | 工作量 | 依赖 |
|------|------|------|--------|------|
| ① 呈现位姿优化器 | 离线算出"UR5 最好插"的工件摆放位姿 | 低（纯软件，**现在就能写**） | 1–2 周 | 无硬件依赖 |
| ② UR10 视觉抓取 | 全局相机找工件→抓取 | 中（用 tag 方案则低） | 1–2 周 | 全局相机 D435 + eye-to-hand 标定 |
| ③ 抓取误差闭合 | 抓完后重定 {H}（视觉链转正） | 低（复用现有代码） | 2–3 天 | 模块② |

## 3. 各模块方法要点

### ① 呈现位姿优化器（论文核心创新点）

- 离线优化，每任务算一次写入 `present_pose`，运行时零开销；
- 候选集：双臂工作空间交集网格 × 孔轴朝向采样；
- 代价函数（加权）：UR5 插入位形可操作度（Yoshikawa w=√det(JJᵀ)）+ 插入轴与重力夹角（越小越好，重力辅助插入）+ 关节限位余量 + 全插入行程 IK 可解无奇异穿越；
- 实现：`workspace_check.py` 从"可达性检查"升级为"打分排序"；UR5/UR10 用解析 IK（闭式解，~100 行 numpy，**离线可跑不需连真机**）。

### ② 视觉抓取（刻意做约束，选档次 1）

- **档次 1（选定）**：工件/托盘贴 AprilTag，全局 D435 俯视检测，`T_tag→抓取点` 由 CAD 给出。几天可通，复用 `tag_locator`；
- 档次 2（备选）：无 tag，深度图平面分割+点云 PCA，1–2 周；
- 档次 3（**明确不做**）：通用 6-DOF 位姿估计（FoundationPose 类）——对论文主张无贡献；
- 需补：全局相机的 **eye-to-hand 标定**（`handeye.py` 改十几行，calibrateHandEye 同函数、变换链方向不同）。

### ③ 抓取误差闭合（关键耦合点）

视觉抓取后工件在爪中位姿有 ±2–5mm 不确定 → 运动学孔位链第三项 T(TCP₁₀←孔) 失信。解法：

```
抓取(有误差) → 摆到优化呈现位姿 → UR5 悬停 → 腕部相机看工件上的 tag
→ 视觉链升级为主链直接给 {H}；运动学链降级为粗校验(一致性阈值 5mm→~15mm)
→ 下游搜孔插入完全不变(视觉链 ±1–2mm 仍在搜孔捕获范围 6mm 内, 误差漏斗成立)
```

代码落点：`dual_arm_manager.refined_H()` 主次链互换 + 阈值调整。

## 4. 论文价值分配（重要认知）

- **创新点在模块①**（固定臂位姿优化是双臂独有自由度，单臂做不了，可够 RA-L）；
- 模块②在论文里只是 pipeline 一环，用最省事的 tag 方案，**别在这花创新预算**；
- 实验设计：倾角 0–60° 扫描 ×（固定呈现位姿 vs 优化呈现位姿）对比成功率/耗时/峰值接触力；
- demo 价值：完整"自动抓取→智能摆位→精准插入"流程视频加分。
- 相关背景：方向 A/B/C 的完整对比、消融实验矩阵见之前的论文讨论（B=同平台 IL vs 传统管线对比，C=拔出在线卡合力辨识）。

## 5. 讨论断点（下次从这里继续）

4. 全局相机装哪：桌面上方龙门架 vs UR10 基座立柱（视野 vs 遮挡权衡）；
