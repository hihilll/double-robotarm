# 方案一详细实施方案：UR5+UR10 双臂协同插拔/装配

> 制定日期：2026-07-03
> 适用任务：① 插座插头的插入/拔出 ② 螺丝对准放入螺母
> 架构：UR10 固定臂（可编程夹具）+ UR5 操作臂（力控插入）
> 技术路线：视觉粗定位 + 导纳控制 + 螺旋搜孔（不依赖学习）

---

## 1. 系统总体架构

### 1.1 硬件清单（BOM）

| 部件 | 推荐型号 | 数量 | 备注 |
|------|---------|------|------|
| 机械臂 | UR5 + UR10（CB3，已有） | 2 | 固件建议升级到 CB3 最新版（≥3.15），RTDE 稳定性更好 |
| 六维力传感器 | ATI Axia80-M20（首选）或 Robotiq FT 300-S | 1 | 装 UR5 腕部。Axia80 以太网 UDP 可到 500 Hz+；FT300-S 只有 100 Hz，可用但需降参（见 §8 风险） |
| 电动夹爪 | Robotiq Hand-E（UR5，行程 50mm 精度高）+ 2F-85（UR10） | 2 | UR5 抓插头/螺丝要求重复精度；UR10 抓插座/螺母要求夹持力 |
| 相机 | RealSense D435/D415 | 1–2 | 1 个装 UR5 腕部（eye-in-hand，必备）；1 个全局俯视（可选，做粗定位） |
| 控制 PC | Ubuntu 22.04，i5 以上，暂不需要 GPU | 1 | 建议加装双网口：一个网段接两台 UR + 力传感器，一个接办公网 |
| 千兆交换机 | 任意 | 1 | UR5、UR10、Axia80、PC 同一网段静态 IP |
| 工装 | 插座固定治具、螺母 V 型夹持指、标定板（AprilTag/棋盘格） | — | 3D 打印即可；夹持指形状与工件配合，消除夹持晃动 |

### 1.2 软件栈

- **语言/库**：Python 3.10 + [`ur_rtde`](https://sdurobotics.gitlab.io/ur_rtde/)（RTDE 125 Hz 实时接口，CB3 支持）+ OpenCV + NumPy/SciPy + pyrealsense2 + 力传感器 SDK（Axia80 用原生 UDP RDT 协议，几十行代码）
- **不建议第一版就上 ROS/MoveIt**：本任务运动都是笛卡尔直线 + 力控微调，不需要复杂规划；纯 Python 直连两台 UR 的 RTDE，链路短、好调试。后续要接 LeRobot 采数据时这套接口也直接复用
- **控制频率**：主力控环 125 Hz（CB3 上限，`dt=8ms`），通过 `servoL()` 流式下发笛卡尔目标；视觉 10–30 Hz 异步跑

### 1.3 坐标系定义

```
{B10}  UR10 基座系 ──┐
{B5}   UR5 基座系  ──┼── 经双臂标定统一到世界系 {W}（取 {B5} 为 {W} 即可）
{T5}   UR5 TCP（夹爪指尖/圆柱尖端）
{FT}   力传感器系（固定于 UR5 法兰）
{C}    腕部相机系（经手眼标定到 {T5}）
{H}    孔坐标系：原点在孔口中心，+Z 指向孔内（插入方向）
```

**{H} 的来源**：UR10 夹持工件时，`{H} = {B10} · FK(q10) · T_夹爪→工件 · T_工件→孔`，其中最后两项由治具 CAD 给出——这是首选，不依赖视觉精度；视觉只负责校验和处理未被 UR10 夹持的场景。

### 1.4 通讯架构（不用 ROS 的实现）

#### UR 控制器的对外接口（任何方案最终都走这些口）

| 端口 | 名称 | 用途 | 频率 |
|------|------|------|------|
| 29999 | Dashboard | 上电/松刹车/加载程序 | 低频 |
| 30001/30002 | Primary/Secondary | 下发 URScript | ~10 Hz |
| 30003 | Real-time | 状态流 + URScript | 125 Hz |
| **30004** | **RTDE** | **双向实时数据交换（本方案采用）** | **125 Hz (CB3)** |

`ur_rtde` 库封装了 RTDE + URScript + Dashboard；ROS 的 UR 驱动内部也是同一套 RTDE 协议，用 ROS 不会获得更快的通道。

#### 进程/线程模型（单 PC、单 Python 进程）

```
                     ┌──────────── 控制 PC (Ubuntu) ────────────┐
 UR5  (192.168.1.10) │  [线程1] 力控主环 125Hz                    │
   ←RTDE 30004──────→│     读UR5状态 → 导纳计算 → servoL          │
 F/T  (192.168.1.30) │  [线程2] F/T采集 500Hz~1kHz                │
   ←UDP RDT─────────→│     UDP流 → 滤波 → 共享缓冲(最新值覆盖)     │
 UR10 (192.168.1.20) │  [线程3] 视觉 10~30Hz（阶段4接入,异步刷{H}）│
   ←RTDE 30004──────→│  [主线程] 任务状态机                       │
 相机 (USB3)─────────→│     编排UR10(moveL阻塞) + UR5原语调用      │
                     └───────────────────────────────────────────┘
```

要点：

1. **两臂 = 两个独立 RTDE 连接**，无需专门"双臂协议"；UR10 只做位置保持，双臂协调退化为状态机内的串行调用
2. **力控环节拍对齐**用 `rtde_c.initPeriod()/waitPeriod()`（与控制器时钟同步，优于 sleep）
3. 线程间共享数据用**最新值覆盖缓冲**（LatestValue），力控环永远读最新帧、不排队
4. 力控线程可设 `SCHED_FIFO` 实时优先级 + Ubuntu lowlatency 内核；无需 PREEMPT_RT
5. 所有设备静态 IP 同一网段，PC 与 UR 之间只过一台交换机

#### 为什么不用 ROS（本项目场景）

1. ROS 解决的问题本项目没有：无避障规划需求（笛卡尔直线+力控微调）、非分布式、无待复用的 ROS 传感器驱动
2. ROS 不解决最难的问题：125 Hz 导纳环必须自己写，放进 ROS 话题多一层延迟抖动；CB3 的 ROS1 驱动已随 ROS1 EOL 停止维护
3. 学界佐证：ALOHA 用 ROS1、Comp-ACT 真机栈基于 ROS，但 2024 后 LeRobot 社区（二期模仿学习要用的生态）完全不依赖 ROS；力控/RL 类论文即便外围用 ROS，力控环也是 ur_rtde/libfranka 直连（参考 ur_ros_rtde 的"ROS2 外壳 + ur_rtde 内核"架构）
4. 迁移路径保留：driver 层抽象好后，将来需要 ROS2（如加移动底盘）只需给 driver 套 node 包装，控制内核不动

## 3. 阶段 1：标定（全部一次性完成，之后锁死硬件）

### 3.1 TCP 标定
UR 示教器四点法标定 UR5 的 TCP 到**夹持物尖端**（插头插针端面中心/螺丝尖端），而不是夹爪指尖。目标精度 ≤0.5 mm。UR10 同理标到夹爪中心。

### 3.2 负载辨识与重力补偿（力控的前提）
1. 控制 UR5 遍历 24–30 个姿态（球面均匀采样腕部朝向），每个姿态静止 1 s，记录 F/T 读数与法兰姿态
2. 最小二乘解出：负载质量 m、质心 r_c、传感器六通道零偏 b
3. 运行时补偿：`F_ext = F_raw − b − G(R)`，其中 G(R) 是当前姿态下 m·g 和 m·g×r_c 的投影
4. **验收**：任意姿态静止时补偿后残差 |F| < 0.5 N、|τ| < 0.05 N·m
5. **注意**：换夹持物（插头↔螺丝）负载会变，为每种工件各存一组参数，任务开始时切换

### 3.3 手眼标定（eye-in-hand）
腕部相机对固定 AprilTag 板采 15–20 组位姿，OpenCV `calibrateHandEye()`（Tsai 或 Daniilidis 法）解 `T_T5→C`。验收：重投影验证点位误差 ≤1.5 mm。

### 3.4 双臂基座标定
方法（推荐 A）：
- **A. 共触点法**：两臂 TCP 各装尖头工具，先后触碰同一组 4–6 个空间点（不共面），SVD 解刚体变换 `T_B5→B10`。成本为零，精度 ~1 mm
- **B. 标定板法**：UR10 夹持 AprilTag 板摆多个位姿，UR5 腕部相机观测，联合解算。可同时校验手眼标定

**验收**：标定后让 UR5 TCP 移动到"UR10 夹持物上某已知点"，实测偏差 ≤1.5 mm。这个偏差决定了搜孔范围的下限。

## 4. 阶段 2：导纳控制底层（核心模块）

### 4.1 控制律

在任务系 {H} 下的导纳模型（平动/转动各三通道独立）：

```
M·ẍ + D·ẋ + K·(x − x_ref) = F_ext
```

离散实现（每通道，dt = 8 ms）：

```python
# 每个 125Hz 周期：
F_h   = R_H.T @ compensate_gravity(F_raw)      # 力变换到 {H} 系并重力补偿
F_h   = deadband(lowpass(F_h, fc=15Hz), 0.5N)  # 滤波 + 死区
acc   = (F_h - D*vel - K*(pos - pos_ref)) / M
vel  += acc * dt
pos  += vel * dt                               # {H} 系下的柔顺偏移
target = pose_ref ⊕ (R_H @ pos)                # 变换回基座系
rtde_c.servoL(target, 0, 0, dt=0.008, lookahead_time=0.1, gain=300)
```

### 4.2 起始参数（调试起点，非最终值）

| 通道 | M | D | K |
|------|-----|------|------|
| 平动 XY（搜孔面内） | 5 kg | 200 N·s/m | 300 N/m（搜孔时降到 0–100） |
| 平动 Z（插入轴） | 5 kg | 300 N·s/m | 0（力控模式：目标恒力代替 K 项） |
| 转动 XY（对准柔顺） | 0.05 kg·m² | 3 N·m·s/rad | 3 N·m/rad（RCC 效果，插入时用） |
| 转动 Z（绕轴） | 0.05 kg·m² | 5 | 20（一般锁住） |

**柔顺中心设在 TCP（工件尖端）**：力矩通道输入用 `τ_tcp = τ_ft + r_ft→tcp × F`，即把传感器力矩折算到尖端。

### 4.3 基本原语（primitive）接口

在导纳层之上封装 5 个原语，后面所有任务用它们拼装：

1. `move_free(pose)` — 纯位置运动（moveL），高刚度
2. `guarded_move(direction, v, F_stop)` — 沿方向低速前进，接触力超阈值即停（接触检测）
3. `hold_force(axis, F_target)` — 沿轴恒力按压，其余通道柔顺
4. `spiral_search(...)` — 见 §5
5. `compliant_insert(depth, F_max)` / `compliant_extract(...)` — 柔顺插入/拔出

### 4.4 安全层（独立于任务逻辑，始终生效）

- |F| > 40 N 或 |τ| > 5 N·m → 立即 `servoStop()` + 抬升 5 mm
- 工作空间盒限位（软件）+ UR 安全平面（示教器配置，硬件级）
- servoL 目标与当前位姿距离 > 5 mm → 判定飞车，急停
- 看门狗：力传感器数据超时 50 ms → 停止
- 调试期示教器速度滑块压到 30%

**验收测试**：① 手推 UR5 末端，各方向顺滑跟随、无振荡；② 对桌面恒力 10 N 按压 30 s，波动 <±1 N；③ 按压中人为横向推工件，恢复稳定。

## 5. 阶段 3：搜孔与插入状态机

### 5.1 状态机（以插头插入为例）

```
IDLE → APPROACH → CONTACT → SEARCH → ALIGN_INSERT → VERIFY → DONE
                     ↓失败       ↓超时                  ↓失败
                  RETRY(≤3) ← RECOVER(抬起5mm重来) ← 
```

各状态实现：

**APPROACH**：`move_free` 到孔口上方 15 mm（沿 {H} −Z 偏移），姿态对齐孔轴。
**CONTACT**：`guarded_move`(+Z_H, v=5 mm/s, F_stop=4 N)。接触后切入力控。
**SEARCH（螺旋搜孔）**：
- Z_H 通道 `hold_force`(6–8 N)，XY 通道叠加阿基米德螺旋轨迹：
  `r(t) = pitch · θ(t)/2π`，pitch = 0.8 mm/圈（< 孔倒角宽度），线速度 4 mm/s，最大半径 6 mm（覆盖 ±3 mm 定位误差 × 安全系数 2）
- **落入判据**（两个条件同时满足）：① Z_H 位置在 100 ms 内突进 >1.5 mm；② Z 向力突降 >50%
- 超时 20 s 未落入 → RECOVER
**ALIGN_INSERT**：
- 落入后停止螺旋，转动 XY 刚度降到 3 N·m/rad（软件 RCC，让插头被孔"扶正"）
- Z_H 恒力 10–15 N 下插，XY 平动低刚度跟随
- **卡阻处理**：若 0.5 s 内深度无进展且横向力 >8 N，叠加小幅摆动（±1° @ 2 Hz，绕横向力矩方向）
- **完成判据**：深度达到目标（插头插入深度，来自 CAD）或 Z 向力达到 F_max 且深度 ≥90%
**VERIFY**：微微回拉 2 N 确认已卡合（插头有卡合力）；松爪、退出。

### 5.2 拔出流程（插头）

1. 视觉/已知位姿 → 夹爪对准插头体，闭合夹持
2. `compliant_extract`：沿 −Z_H 力斜坡拉拔（5 N 起，每秒 +5 N，上限 30 N），XY 平动和转动 XY 低刚度（自动顺着插针方向退）
3. 若 25 N 仍未动 → 叠加 ±1.5° 摆动同时拉
4. **拔出判据**：Z_H 位移超过插针长度 + 力骤降

### 5.3 螺丝入螺母的差异点

- UR10 用 V 型指夹持螺母（或夹持带螺母的工件），螺母轴线位姿由治具 CAD 给出
- UR5 夹持螺丝头部（Hand-E 平行指 + V 槽指尖），TCP 标到螺丝尖端
- 流程与插头相同，参数更保守：接触力 3 N、搜孔恒力 4–5 N（螺纹尖端接触面小，力大会滑牙/打滑）、螺旋 pitch 0.5 mm
- 落入判据的深度阈值按螺丝导向段长度调整；**只做放入**：落入 + 下插 1–2 个牙深即算成功
- 若后续要拧入：UR5 绕 Z_H 恒速旋转 + Z 向小恒力（力位混合），检测"力矩上升+每转进给=螺距"判定啮合——留作二期

## 6. 阶段 4：视觉定位

**分两档实现，先 A 后 B：**

**A. AprilTag 方案（第一版，1 周内可用）**
- 在插座治具/螺母治具上贴 AprilTag（tag36h11，边长 20–30 mm），离线量好 `T_tag→孔`
- 腕部相机在悬停位姿检测 tag → 算出 {H} → 精度可达 ±1 mm
- 用途：即使 UR10 夹持位姿已知，tag 也用来吸收夹持重复性误差

**B. 无标记方案（二期）**
- 插座：RGB 圆孔检测（Hough/椭圆拟合）+ 深度图端面法向 → 5-DOF 孔位姿
- 螺母：同理检测内孔圆
- 达到 ±2 mm 即可，剩余误差交给搜孔

**粗到精流程**：全局相机（若有）粗定位 → UR5 移到上方 → 腕部相机近距离（15–20 cm）精定位 → 刷新 {H} → 进入状态机。

## 7. 阶段 5：双臂集成与任务流程

### 7.1 UR10 侧逻辑（简单，纯位置控制）

1. `pick`: 从料位抓取插座/螺母工件（位置示教或全局相机定位）
2. `present`: 移动到预定呈现位姿（工作空间中双臂都舒展、UR5 无奇异的区域，提前在仿真/示教器里选好），**保持不动**（关节位置锁定即可，CB3 位置保持刚度足够）
3. 任务完成后 `place` 放回
- UR10 呈现位姿一旦确定就写入配置文件，{H} 随之确定

### 7.2 主流程编排

单 PC 上一个进程、两个 RTDE 连接。UR10 动作与 UR5 动作串行即可（无需同步运动）：

```
主状态机：
  UR10.pick → UR10.present → (视觉刷新{H}) → UR5.插入状态机
  → UR5 退出 → UR10.place → 循环
```

### 7.3 工作空间布置建议

- 两臂基座间距 1.2–1.5 m，呈 90–120° 相对布置；呈现区设在两臂工作空间交集的中部偏 UR5 一侧
- UR5 在呈现区的位形避开腕部奇异（wrist singularity：q5 ≈ 0）——布置前用 `ur_rtde` 的 IK 或 RoboDK 扫一遍
- 插入方向第一版选竖直向下（{H} Z 轴 = −Z 世界），最简单；斜孔扩展见调研文档 §四

## 8. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| FT300-S 只有 100 Hz | 导纳环相位滞后，高刚度下易振荡 | 优先买 Axia80；若用 FT300：D 加大 50%、接触速度减半 |
| CB3 servoL 抖动/延迟 | 力控品质下降 | gain 调 200–500 之间试；确保 PC 与 UR 直连同一交换机，控制循环用 `rtde_c.initPeriod()/waitPeriod()` 对齐节拍 |
| 插头夹持打滑 | 插入力传不上去 | 指尖做仿形 3D 打印 + 防滑垫；VERIFY 阶段检测夹持位移 |
| 插座卡合力大于拔出力上限 | 拔不出 | 实测卡合力后调上限；治具增强插座固定刚度 |
| 双臂标定漂移（撞击/温度） | 搜孔起点偏出范围 | AprilTag 在线校验兜底；每天开机跑一次快速验证点 |
| 螺纹滑牙 | 螺丝任务失败 | 搜孔力压到 3–5 N；选带倒角/导向段的螺丝先验证 |

## 10. 评测方案（阶段 6）

- **随机化协议**：UR10 呈现位姿在 ±3 mm / ±3° 内随机扰动（模拟定位误差），每任务 30 次
- **指标**：成功率、平均耗时、搜孔耗时占比、峰值接触力、失败模式分类（未接触/搜孔超时/卡阻/误检成功）
- **目标**：插头插入 ≥90%，拔出 ≥95%，螺丝入螺母 ≥85%；峰值力不超限；单循环 <40 s
- 失败案例的日志逐条复盘 → 反向调整参数表，这个闭环跑 2–3 轮
