> 创建：2026-07-03 ｜ 配套：[项目进度与代码计划.md](项目进度与代码计划.md)（任务编号 # 与其一致）
> 本手册回答三个问题：**装什么环境、现在能测什么怎么测、到货后按什么顺序测**。

## 1. 环境要求总览

| 环境 | 用途 | 系统 | 何时需要 |
|------|------|------|---------|
| A. 开发机（现在这台 Windows） | 离线自检、改代码、看日志 | Windows 10/11 + Python 3.10+ | 现在 |
| B. 控制 PC | 连接两臂/传感器跑真机 | **Ubuntu 22.04** + Python 3.10+ | 到货前装好（也可先用现有任意 Ubuntu 机器测 #13–15） |
| C. URSim 仿真器（可选） | 采购期把 RTDE 链路测干净 | VirtualBox 虚拟机 | 可选 |

> 真机控制必须用 Ubuntu：`servoL` 125Hz 流控对时序敏感，Windows 下抖动大且 RealSense/串口驱动折腾。开发机只跑离线测试。

## 2. 环境配置步骤

### 2.1 开发机（Windows，现在就做）

```powershell
python --version          # 需要 ≥3.10
cd D:\double_robotarm
python -m venv .venv
.venv\Scripts\activate
pip install numpy scipy pyyaml           # 离线自检只需这三个
# 可选装全量（ur-rtde 有 Windows wheel; pupil-apriltags 装不上可跳过, 不影响自检）:
pip install -r requirements.txt
```

验证：`python scripts/_selftest_wiring.py` 输出 `all wiring self-tests PASSED` 即环境 OK。

### 2.2 控制 PC（Ubuntu 22.04，到货前装好）

```bash
# 1) 系统与工具
sudo apt update && sudo apt install -y python3-venv python3-pip git net-tools

# 2) 项目与依赖
#    把 D:\double_robotarm 整个目录拷到 ~/double_robotarm (U盘/git均可)
cd ~/double_robotarm
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
pip install pyrealsense2          # 相机到货后; D405 需要 librealsense ≥2.51

# 3) 串口权限(夹爪 Modbus-RTU 用)
sudo usermod -aG dialout $USER    # 重新登录生效; 夹爪插入后 ls /dev/ttyUSB*

# 4) (推荐)低延迟内核, 降低 servoL 抖动
sudo apt install -y linux-lowlatency && sudo reboot
```

依赖包名对照（pip 名 ≠ import 名）：`ur-rtde`→`rtde_control/rtde_receive`；`pupil-apriltags`→`pupil_apriltags`；`opencv-python`→`cv2`。

验证：同样跑 `python scripts/_selftest_wiring.py` 全绿。

### 2.3 网络规划与配置

所有设备静态 IP、同一网段、一台千兆交换机：

| 设备 | IP | 配置位置 |
|------|-----|---------|
| 控制 PC | 192.168.1.5 | Ubuntu 设置→网络→手动 IPv4（掩码 255.255.255.0，网关留空） |
| UR5 | 192.168.1.10 | 示教器：设置机器人→网络→静态地址 |
| UR10 | 192.168.1.20 | 同上 |
| 力传感器 | 192.168.1.30 | 传感器 Web 配置页（ATI 出厂默认 192.168.1.1，**先直连改成 .30** 再入网；国产按手册） |

连通性验证（PC 上）：

```bash
ping 192.168.1.10 && ping 192.168.1.20
nc -zv 192.168.1.10 30004        # RTDE 端口通 = ur_rtde 能连
```

改完 IP 记得同步 `config/robots.yaml`。

### 2.4 UR 示教器设置（CB3，两台都做）

1. **固件**：设置→关于，建议升级到 CB3 最新（≥3.15），RTDE 稳定性更好；
2. **安全平面**：安装设置→安全→平面，在桌面下方和双臂交界处各设一个（硬件级兜底）；
3. **速度滑块**：整个调试期压到 **30%**；
4. **TCP/负载**：示教器里保持默认 0（代码里通过 `setTcp` 下发，避免两处不一致）；
5. **运行状态**：跑 RTDE 控制时示教器**不要运行任何本地程序**（空闲在主界面即可），否则 `RTDEControlInterface` 会连接失败。

### 2.5 （可选）URSim 仿真器——采购期提前测 RTDE 链路

1. UR 官网下载 **URSim CB3**（Non-Linux 版，VirtualBox 镜像，选与真机相同固件版本）；
2. VirtualBox 导入 → 网络改成"桥接网卡" → 启动后在虚拟机终端 `ifconfig` 查 IP；
3. 把该 IP 临时填进 `robots.yaml` 的 ur5.ip → 跑 `check_comm.py`、`jog_test.py` 均可通；
4. **局限**：无力传感器数据、接触动力学不真实——只能测通讯和运动指令，力控相关一律等真机。

## 3. 现阶段测试（零新硬件）

### 3.1 离线自检（开发机，现在就跑）

```powershell
python scripts/_selftest_wiring.py
python scripts/_selftest_calib.py
```

**预期**：`all wiring self-tests PASSED` / `payload solver OK ... kabsch OK ... all imports OK`。
**用途**：以后每次改代码或配置，先跑这两个再上真机。

### 3.2 任务#13：双臂通讯检查（需要两臂上电入网，机器人不动）

前置：§2.3 网络配好、两臂开机松刹车、示教器空闲。

```bash
python scripts/check_comm.py
```

**预期**：两臂各打印 TCP 位姿、关节角、保护性停止=False；末行"阶段0通讯检查通过 ✓"（力传感器部分未到货会报错，属正常，注释掉该段或到货后补测）。
**常见问题**：连接超时→查 IP/网线/是否松刹车；`control script running` 类错误→示教器上停止本地程序。

### 3.3 任务#14：点动测试（⚠ 首次让机器人动，速度滑块 30%，手扶急停）

```bash
python scripts/jog_test.py --arm ur5
python scripts/jog_test.py --arm ur10
python scripts/jog_test.py --arm ur10 --present   # present_pose 初值试运行
```

**预期**：各轴 ±2cm 平稳往返，无保护性停止。
**检查点**：运动方向与你对基座系的理解一致（站在机器人正面确认 X/Y 朝向），后面标定和治具布置都依赖这个直觉。

### 3.4 任务#15：呈现区可达性扫描（不动机器人）

```bash
python scripts/workspace_check.py --center 0.45 0.0 0.25 --size 0.30 --step 0.05
```

center 先目测选在两臂都够得着的区域中部偏 UR5 一侧。**预期**：可用率 100%；否则按输出的问题位形移动 center 重扫。定稿后把对应的 UR10 呈现位姿写入 `robots.yaml ur10.present_pose`。

**→ 三项全过 = 里程碑 M0 ✅（进度文档 §5 打勾）**

## 4. 到货后测试（严格按顺序）

### 4.0 硬件安装与配置回填（半天）

1. **力传感器**：转接板装 UR5 法兰（注意安装方向做记号）→ 网线入交换机 → Web 页改 IP 为 .30 → 把标定单上的 counts_per_force/torque 填入 `robots.yaml`，安装朝向填 `R_tcp_sensor`、原点到 TCP 矢量填 `p_sensor_to_tcp`（按转接板 CAD）；
2. **夹爪**：UR5/UR10 各装一只，USB-RS485 接 PC，`ls /dev/ttyUSB*` 确认端口号填入 yaml；**对照到货手册核对 `drivers/gripper.py` 顶部寄存器映射**（⚠ 项之一）；
3. **相机**：D405 装 UR5 腕部支架，`rs-enumerate-devices` 查序列号填 yaml；
4. 重跑 `_selftest_wiring.py` + `check_comm.py`（此时力传感器帧率应达标：年龄中位数 < 1000/输出频率 ms）。

### 4.1 任务#18：TCP 标定（示教器，每种工件一组）

装上夹爪夹持工件 → 示教器：安装设置→TCP→位置向导（四点法：4 种姿态触同一固定尖点）→ 结果抄入 `robots.yaml tcp_offset`（示教器里不保存，恢复为 0）。误差 >0.5mm 就重做，这个精度直接决定搜孔范围。

### 4.2 任务#16：负载辨识（每种夹持状态各跑一次）

```bash
python calib/payload_id.py --payload none    # 仅夹爪
python calib/payload_id.py --payload plug    # 夹持插头(先#18)
python calib/payload_id.py --payload screw
```

**预期**：打印 `达标 ✓`（残差 |F|<0.5N |τ|<0.05Nm），输出的 yaml 行粘贴进 `robots.yaml payloads`。
**不达标排查**：夹持松动→重夹；残差集中在力矩→`R_tcp_sensor` 安装矩阵填错。

### 4.3 任务#17：力符号校核（⚠ 全项目最重要的半小时）

写个两行脚本或用 `check_comm.py` 持续打印补偿后 wrench，然后**手动沿传感器各轴施力**，逐格填表核对：

| 操作 | 预期读数 | 实测 | 结论 |
|------|---------|------|------|
| 沿传感器 +X 推 | Fx 为 __（按传感器手册定义填）| | |
| 沿 +Y / +Z 推、绕各轴拧 | 同理 6 项 | | |

如与 `control/primitives.py` 头注释的约定相反：只改 `wrench_H()` 一处符号，**不要**到处改。此项不做对，后面导纳会"越推越顶"，且有安全风险。

### 4.4 任务#21：手推柔顺（力控第一次上电）

```bash
python scripts/demo_admittance.py --payload none
```

⚠ 速度滑块 30%、手扶急停。**调参顺序**（改 `admittance.yaml hand_push_demo` 与 `ur_arm.py servo_l` 的 gain）：

| 症状 | 处理 |
|------|------|
| 高频振荡/异响 | gain 降（300→200）；D 加大 50% |
| 推不动/迟钝 | D 减小；M 减小 |
| 松手后漂移 | 力零偏没除净→重跑 #16；或加大死区 |
| 报"数据超时" | 传感器帧率不足→查网络/降 lpf 截止 |

**预期**：手推各方向顺滑跟随、松手即停、无振荡。

### 4.5 任务#22：恒力按压 → 里程碑 M2


**预期**：`达标 ✓ (M2)`（最大偏差 <1N）。期间侧推工件应稳定恢复。

### 4.6 任务#19：双臂标定 → 里程碑 M1

两臂各装尖头工具（TCP 先按 #18 标到针尖），呈现区周围布 5 个不共面触点（间距 >15cm）：

### 4.7 任务#24：治具上桌 + {H} 实测

插座治具固定桌面（先不上 UR10，降变量）。UR5 装针尖工具，拖动触**孔口边缘 3 点**记录 TCP 位置 → 三点平均 = 孔口中心 (cx,cy,cz)。竖直孔的 {H} = `[cx, cy, cz, 3.1416, 0, 0]`（+Z 指向孔内）。

### 4.8 任务#25–28：搜孔插入调参 → 里程碑 M3（预留 1–2 周）

```bash

# 逐参数调优: 按"search_force → spiral_pitch → spiral_v"顺序改 admittance.yaml, 每改一个跑5次
# 每次失败: python scripts/plot_log.py logs/<最新>.h5   ← 看力曲线定位原因
```

**验收**：成功率 ≥90% → **M3 ✅（项目成败关键点）**。典型问题：搜孔擦过不落入→TCP 误差大或 drop_dz 阈值紧，螺距减半；落入后卡死→转动刚度 kr 还太高。
螺丝任务同法：`--task screw_place`（低力参数组已配好）。

## 5. 故障排查速查表

| 现象 | 最可能原因 | 处理 |
|------|-----------|------|
| RTDEControl 连接失败 | 示教器在跑本地程序 / IP 错 / 未松刹车 | 停程序；ping + nc 30004 |
| 一动就保护性停止 | 速度/加速度超限、接近奇异 | 降速；workspace_check 重查位形 |
| servoL 顿挫 | 网络抖动 / PC 负载高 | 直连交换机；关无关进程；lowlatency 内核 |
| "力传感器数据超时"急停 | 传感器断流 / IP 冲突 | 重启传感器；查 Web 页流设置 |
| 导纳振荡 | gain 高 / D 小 / 传感器帧率低(FT300) | §4.4 表；FT300 则 D×1.5、接触速度减半 |
| 夹爪无响应 | 端口号/波特率/寄存器映射不对 | ls /dev/ttyUSB*；对手册核对 gripper.py |
| 搜孔总擦过 | TCP 标定差 / 双臂标定漂移 | 重做 #18/#19；tag 在线校验兜底 |
| Tag 检出率低 | 光照/尺寸填错/距离太近失焦 | 补光；实测 size；悬停距离 15–20cm |

## 6. 安全须知（每次真机测试前过一遍）

1. 速度滑块 30%，一人操作一人手扶急停（至少调参期）；
2. 首次跑任何新原语/新参数：先空中无接触跑一遍看轨迹，再上工件；
3. 安全平面（§2.4）在第一次点动前必须配好；
4. 换工件/夹爪后：重跑对应 payload 辨识，否则重力补偿失效＝虚假接触力；
5. 任何硬件挪动（相机、传感器、两臂基座）→ 对应标定作废重做。
