## 第 1 课　总体思想：误差漏斗

### 学习目标
理解整个系统为什么这样设计；学会用"误差预算"审视任何精密操作系统。

### 1.1 问题的本质矛盾

插头与插孔的配合间隙约 0.1–0.5 mm，而系统中没有任何单一环节能达到这个精度：

| 环节 | 能提供的精度 |
|------|------------|
| 双臂标定 + UR10 运动学算孔位 | ±1.5–3 mm |
| 视觉（AprilTag） | ±1–2 mm |
| UR 重复定位精度 | ±0.1 mm（但绝对精度差得多） |

**结论：靠"算准了再插"这条路走不通。** 所有精密装配系统（包括人手）的共同答案是：用**接触本身**作为最后的信息来源和约束。

### 1.2 误差漏斗

把任务设计成逐级收窄的漏斗，每一级只需把误差压进**下一级的捕获范围**：

```
运动学/视觉定位   误差 ≤3 mm ──┐
                              ↓ 被螺旋搜孔捕获（搜索半径 6 mm ≥ 2×3 mm）
螺旋搜孔         误差 ≤倒角宽度 ──┐
                              ↓ 被孔口倒角捕获（螺距 0.8 mm < 倒角宽度）
落入孔口         斜卡状态 ──┐
                              ↓ 被孔壁几何约束捕获（柔顺插入/软件 RCC）
完全插入         误差 = 0
```

**设计规则**：上一级的最大输出误差 × 安全系数 ≤ 下一级的捕获窗口。项目里所有验收数字（双臂标定 1.5 mm、搜索半径 6 mm、螺距 0.8 mm）都由此倒推。

### 读代码
`项目进度与代码计划.md` §3 的验收标准列 —— 逐项想一想每个数字对应漏斗的哪一级。

1. 如果换一种没有倒角的孔，漏斗哪一级断了？有什么补救思路？
2. 视觉精度若提升到 ±0.3 mm，可以省掉哪一级？值得吗？

## 第 2 课　基础语言：位姿、坐标系与力旋量

### 学习目标
掌握本项目所有数学的"词汇表"：pose6、旋转向量、坐标变换、wrench 及其两种变换。

### 2.1 位姿表示：pose6 与旋转向量

UR 约定位姿为 `[x, y, z, rx, ry, rz]`：前三个是平移（米），后三个是**旋转向量**（axis-angle）——方向是转轴、模长是转角（弧度）。例如 `[π, 0, 0]` 表示绕 X 轴转 180°。

优点：无奇异参数化最少（3 个数）；缺点：不能直接相加复合。所以代码里所有复合运算都先转成旋转矩阵：

```
pose_to_Rp(pose)  →  (R 3×3, p 3×1)
compose(a, b) = T_a ∘ T_b:  R = R_a·R_b,  p = p_a + R_a·p_b
```

**算例**：UR10 基座在世界系 `[1.3, 0, 0, 0, 0, π]`（对面放置，绕 Z 转 180°），UR10 报告自己 TCP 在 `[0.5, 0, 0.4, ...]`。TCP 的世界系 x 坐标 = 1.3 + cos(π)·0.5 = **0.8 m**——绕 Z 转 180° 后，UR10 的 +X 指向世界 −X。（这正是 `_selftest_wiring.py` 第 3 段验证的东西。）

### 2.2 本项目的六个坐标系

```
{W}=UR5基座 ── {B10} UR10基座（双臂标定）
   ├─ {T5} UR5 TCP（运动学+TCP标定）
   │    ├─ {FT} 力传感器（安装矩阵 R_tcp_sensor）
   │    └─ {C} 腕部相机（手眼标定）
   └─ {H} 孔系：原点在孔口中心，+Z 指向孔内
```

**{H} 是全项目的支点**：控制、搜索、插入全在 {H} 下表达，"孔是斜的"只是 {H} 的姿态不同，代码零改动。

### 2.3 力旋量（wrench）及其两种变换

wrench = `[Fx, Fy, Fz, Tx, Ty, Tz]`（力 + 力矩）。它有两种容易混淆的变换：

1. **换坐标系（纯旋转）**：`F' = R·F, T' = R·T` —— 只是换了描述语言，物理量没变；
2. **换参考点（搬移）**：`T' = T − r×F` —— 力矩本身依赖于"绕哪个点"，从传感器原点搬到工件尖端要加上力臂效应。

**算例**：尖端受纯侧向力 Fx=10 N，传感器在尖端上方 0.12 m 处。传感器读到的力矩 Ty = 10×0.12 = 1.2 N·m；折算回尖端后 Ty′ = 1.2 − 0.12×10 = **0**——尖端受纯力、绕尖端没有力矩，物理直觉吻合。折算的意义见第 5 课"柔顺中心"。

## 第 3 课　标定四件套：用不变量钉住坐标系

### 学习目标
理解四项标定共同的方法论——构造物理不变量 + 超定最小二乘；能独立解释每项的原理和验收逻辑。

### 3.0 共同方法论

每项标定都在利用一个**物理不变量**：

| 标定 | 不变量 | 求解工具 |
|------|--------|---------|
| TCP | 尖端触的是同一个空间点 | 线性最小二乘 |
| 负载 | 重力大小方向恒定 | 两段线性最小二乘 |
| 双臂 | 触的是同一组点 | SVD（Kabsch） |
| 手眼 | 标定板固定不动 | AX=XB（Tsai） |

采多组数据 → 超定方程 → 最小二乘。**多组数据不是为了保险，是为了让噪声被平均掉、让解唯一。**

### 3.1 TCP 四点法

未知量：尖端在法兰系的偏移 t。4 种姿态触同一点：`R_i·t + p_i = c`（c 也未知）。4 组数据 12 个方程解 6 个未知数。姿态差异越大，方程组条件数越好——这就是操作规程要求"姿态尽量拉开"的数学原因。

### 3.2 负载辨识（重点，力控的地基）

传感器读数 = 零偏 + 重力投影 + 接触力。静止无接触时：

```
F = b_F + m·(Rᵀg)             ← 未知: m, b_F
T = b_T + c×(m·Rᵀg)           ← 未知: c, b_T
```

对未知量全部线性！两段求解：先由力方程解 (m, b_F)，代入力矩方程解 (c, b_T)（叉乘 `c×v = −skew(v)·c` 对 c 线性）。

**算例**：工具 m=1 kg 竖直朝下时 Fz 读数含 −9.81 N；转到水平时这 9.81 N 挪到了 Fx/Fy。不补偿的话，导纳控制在水平姿态会以为有人用 ~10 N 推它——**这就是斜孔任务必须先做负载辨识的原因**。

验收残差 <0.5 N 的另一层含义：这是系统的**力觉底噪**，第 6 课的接触阈值（4 N）必须显著高于它才不误触发。

### 3.3 双臂标定（Kabsch）

两臂触同一组点 → 两个坐标系下的对应点集 P、Q → 求刚体变换使 P ≈ R·Q + t。闭式解：去质心 → H = ΣqᵢpᵢᵀT → SVD(H) → R = V·D·Uᵀ（D 修正反射）→ t = p̄ − R·q̄。

**为什么用留一交叉验证而不是拟合残差**：4 个点拟合 6 自由度变换几乎必然"拟合得很好"，残差偏乐观；留一验证测的是**没参与拟合的点**的预测误差，才是真实精度。

### 3.4 手眼标定（AX = XB）

机器人两位姿间的运动 A（运动学给出）与相机看到的板子相对运动 B（视觉给出）通过未知的手眼变换 X 关联：AX = XB。验证同样用不变量：板子不动 ⇒ 各样本算出的"板子在基座系的位置"应重合，散布即误差。

### 读代码
`calib/payload_id.py` 的 `solve_payload()`、`calib/dual_arm_calib.py` 的 `kabsch()` —— 再看 `scripts/_selftest_calib.py` 如何用合成数据验证它们（这个"造已知真值→加噪→求解→对比"的自检模式值得学会，任何估计算法都该这么测）。

### 练习

## 第 4 课　力感知管线：从原始读数到可用信号

### 学习目标
掌握 8 ms 周期内力数据的四道工序及每道的存在理由。

```
原始 wrench(传感器系)
  → ① 低通滤波(15Hz)      滤机械振动/电噪; 截止太高→振荡, 太低→接触响应迟钝
  → ② 重力补偿             扣零偏+自重投影(第3课参数); 剩纯接触力
  → ③ 力矩折算到尖端       T′=T−r×F; 把"柔顺中心"放到工件尖端
  → ④ 旋转到 {H}           后续控制不再关心孔的朝向
干净接触力(孔系, 绕尖端)
```

**一阶低通的实现**：`y += α·(x−y)`，α = 1−e^(−2πf_c·Δt)。f_c=15 Hz、传感器 500 Hz 时 α≈0.17——每帧向新值靠近 17%，兼顾平滑与延迟（群延迟约 10 ms，相对 8 ms 控制周期可接受；这也是为什么 100 Hz 的传感器很勉强——滤波后延迟占了好几个控制周期）。

**死区（0.5 N）**：底噪之内的读数直接置零，否则导纳控制会让机械臂"追着噪声抖"。

### 读代码
`drivers/ft_sensor.py`（①②）、`control/primitives.py` 的 `wrench_H()`（③④，注意四道工序的顺序为什么不能换——补偿必须在传感器系做，因为零偏是传感器固有的）。

### 自测
1. 为什么重力补偿要在旋转到 {H} **之前**做？
2. FT300（100 Hz）相比 Axia80（500 Hz+）在这条管线上具体差在哪一环？

## 第 5 课　导纳控制：给位置机器人装上"软件弹簧"

### 学习目标
理解导纳控制的物理图像、离散实现、参数整定逻辑；理解"逐通道设参"如何构成柔顺策略。

### 5.1 为什么是导纳而不是阻抗

- **阻抗控制**：测运动 → 输出力矩。需要关节力矩接口（Franka 有，UR 没有）；
- **导纳控制**：测力 → 输出位置修正。只需位置接口 + 外置力传感器 —— UR 唯一可行的路。

### 5.2 虚拟动力学与离散实现

在 {H} 每个通道虚构一个弹簧-质量-阻尼系统：`M·ẍ + D·ẋ + K·x = F_ext − F_ref`

每 8 ms 做一次显式积分：

```
a = (F − D·v − K·x)/M ;  v += a·Δt ;  x += v·Δt   (限幅)
```

x 是柔顺偏移，叠加到参考位姿发 servoL。**F_ref 的妙用**：K=0 的通道给 F_ref = −10 N，平衡点变成"接触反力恰好 10 N"——恒力按压不需要单独的控制器，只是导纳的一种参数配置。

**手算算例**（体会参数量纲）：M=5 kg、D=200、K=0，突加 F=10 N：
- 第 1 帧：a=2 m/s²，v=16 mm/s，x≈0.13 mm
- 稳态：a=0 ⇒ v_∞ = F/D = **50 mm/s** —— K=0 时导纳是个"速度源"，力大小决定移动速度。这解释了为什么恒力搜孔时工件会持续贴着表面走。

### 5.3 参数的调试语义

| 参数 | 物理含义 | 调试症状 |
|------|---------|---------|
| K | 回中的劲；K=0 变力控 | 太高→接触硬碰硬; 太低→定不住 |
| D | 耗能防振 | 太低→振荡(最常见); 太高→迟钝 |
| M | 惯性感 | 太低→对噪声敏感; 太高→响应慢 |
| servoL gain | 底层跟踪紧度 | 太高→高频抖; 太低→软塌 |

**调参顺序 gain→D→M 的原因**：内环（servoL）不稳，外环怎么调都是白调。

### 5.4 六通道参数表 = 柔顺策略

同一时刻不同通道干不同的事，例如插入阶段：

```
平动 z: K=0 + F_ref=−12N   恒力推进
平动 xy: K=300              中等弹簧, 跟随孔壁
转动 xy: K=3 (超软!)         ← 下一课的软件 RCC
转动 z:  K=20               锁住, 防绕轴打转
```

**策略切换只是换参数表**——这就是 `admittance.yaml` 按模式组织参数的原因。

### 读代码
`control/admittance.py`（30 行核心积分）、`config/admittance.yaml`（对照 5.4 理解每个模式）。

## 第 6 课　接触策略：搜孔、对准与插入

### 学习目标
掌握状态机每个阶段的方法原理，特别是螺旋搜孔的几何条件、双条件判据和软件 RCC。

### 6.1 guarded move：把位置不确定换成接触事件

低速（5 mm/s）下压 + 监测运动方向反力 >4 N 即停。之后一切以**接触面**为基准——垂直方向的定位误差就此归零。速度必须低的原因：从检测到停下有约 2–3 个控制周期延迟，5 mm/s 下过冲只有 ~0.1 mm。

### 6.2 螺旋搜孔

**配方**：Z 通道恒力 6–8 N（第 5 课的 K=0+F_ref）压着表面 + XY 画阿基米德螺旋 r = pitch·θ/2π。

三个参数的确定逻辑（全部来自误差漏斗）：
- **最大半径 6 mm** = 上游定位误差 3 mm × 安全系数 2；
- **螺距 0.8 mm < 孔倒角宽度**：保证螺旋线不可能跨过孔口（相邻两圈的间隔小于捕获窗口）;
- **线速度 4 mm/s**：太快会因为落入检测的 100 ms 窗口而冲过孔。

**落入判据双条件**：100 ms 内 Z 突进 >1.5 mm（掉下去）**且** Z 反力降 50%（脚下空了）。单一条件的失效模式：只看位置→表面台阶/毛刺误报；只看力→摩擦波动误报。与运算同时挡掉两类误报。

### 6.3 软件 RCC：让孔自己完成对准

落入后工件斜卡在孔口（两点接触）。此时孔壁的接触力天然形成**扶正力矩**——工件歪向哪边，孔壁顶它另一边。把转动 XY 刚度降到 3 N·m/rad，导纳控制就会顺着这个力矩把工件转正：**对准是孔的几何形状完成的，控制器只是"不阻拦"。**

两个前提缺一不可：
1. **柔顺中心必须在工件尖端**（第 4 课的力矩折算）：绕尖端转，尖端不动、尾巴摆——越转越正；若绕法兰转，转动会带着尖端扫出孔口——越"顺"越歪；
2. 转动刚度必须远低于扶正力矩对应的等效刚度，否则弹簧劲比扶正劲大，卡死。

### 6.4 卡阻摆动与力斜坡拔出

- **卡阻**（0.5 s 无进展 + 横向力 >8 N，典型是两点接触自锁）：沿横向力矩方向叠加 ±1°@2 Hz 摆动——把静摩擦变动摩擦，打破摩擦锥自锁。和你手动拔紧插头时晃两下同理；
- **拔出用力斜坡**（5 N 起每秒 +5 N）而非定值：卡合力未知，斜坡=用最小必要力完成任务，且力骤降瞬间的读数就是实测卡合力（顺便完成了测量）；
- **螺丝任务同构不同参**：螺纹牙是斜面、接触面积小，力大即滑牙——接触 2.5 N/搜孔 4 N/螺距 0.5 mm，方法一个字不改。

`control/primitives.py` 的三个原语主环（对照本课逐段读）、`tasks/insertion.py` 的状态转移与重试逻辑。

## 第 7 课　双臂协同与视觉：信息架构

### 学习目标
理解"UR10=可编程夹具"的设计哲学与双测量链互校验的可靠性思想。

### 7.1 非对称协同：刻意不做"真双臂"

UR10 全程纯位置控制：抓取→呈现→定住→放回。它的价值不在运动而在**信息**——它夹着工件时，孔位可以纯计算得出：

```
{H} = T(W←B10) ∘ T(B10←TCP₁₀) ∘ T(TCP₁₀←孔)
      双臂标定      UR10运动学       治具CAD
```

**不用视觉就有 ±1.5 mm 的孔位**。这是双臂配置对"单臂+固定夹具"的本质优势：夹具位姿是活的、可编程的、且永远已知。副产品：系统复杂度从"双臂协同"降为"单臂插孔+会动的夹具"——这是 2–3 个月能落地的前提。斜孔扩展也在这里：让 UR10 把工件转个角度，{H} 链路自动给出新孔位。

### 7.2 视觉的角色：冗余校验而非唯一来源

AprilTag 精定位（PnP 解位姿 + 多帧中位数滤波）提供第二条独立测量链。执行前比较两条链的 {H}：偏差 >5 mm → 报警拒执行。

**可靠性思想**：两条链的误差源完全独立（一条错在标定漂移/夹持滑动，一条错在 tag 贴歪/光照），单点故障不再直接变成撞机，而是变成一次可诊断的报警。中位数滤波（而非均值）的理由：视觉的失效模式是偶发野值（误检），中位数对野值鲁棒。

### 读代码
`tasks/dual_arm_manager.py` 的 `kinematic_H()` / `refined_H()`；`perception/tag_locator.py` 的坐标链与中位数逻辑。

2. 一致性检查阈值 5 mm 是怎么和搜孔半径 6 mm 呼应的？

## 第 8 课　评测：把调参变成受控实验

### 学习目标
掌握随机化评测协议与失败驱动的参数迭代闭环。

### 8.1 三条纪律

1. **随机化**：呈现位姿注入 ±3mm/±3° 均匀扰动。不随机化的 20 连胜只证明你调好了**那一个点**——泛化性为零；
2. **失败模式分类**：{未接触 / 搜孔超时 / 卡阻 / 误检成功}，每类指向不同参数（搜孔超时→查 TCP 标定与螺距；卡阻→查转动刚度与摆动阈值）。分类把调参从瞎试变成定向修复；
3. **单参数迭代 + 全量回归**：一次改一个参数 → 重跑 30 次评测。改两个参数后变好了，你不知道是哪个的功劳，更不知道另一个是否埋了雷。

### 8.2 日志是方法的物质基础

125 Hz 全量日志（力/位姿/状态机状态）让每次失败可回放：搜孔超时的力曲线能看出是"擦过了孔"（有短暂力扰动）还是"根本没到孔附近"（一路平滑）——两者的修法完全不同。**没有日志的调参是玄学，有日志的调参是工程。**

### 读代码
`scripts/evaluate.py` 的统计与报表、`scripts/plot_log.py`、`common/logger.py`（注意"主环只入队、写盘在独立线程"——实时系统里日志绝不能阻塞控制环）。

### 自测
1. 为什么扰动加在 UR10 呈现位姿上，而不是直接加在 {H} 上？（提示：哪个更接近真实误差来源）
2. "误检成功"这类失败最危险，为什么？用什么手段压制它？

[标定] 不变量+最小二乘 → 坐标系全部钉死, 力觉底噪 <0.5N

[力管线] 滤波→重力补偿→折算尖端→旋到{H} → 干净接触力

[导纳] 虚拟M-D-K逐通道 → 位置机器人可编程软硬; K=0+F_ref=恒力
   ↓

   ↓

## 附录 B　自测题参考答案（要点）

**1 课**：① 断在"搜孔→落入"级，补救：更尖的工件导向头（把倒角做在工件上）/提高上游精度/改力觉盲搜判据；② 可省搜孔级，但通常不值——视觉达到 0.3 mm 需要昂贵硬件和苛刻光照，而搜孔只花几秒钟。
**2 课**：① 旋转向量不满足加法交换/结合（旋转群非线性）；② 力矩折算方向错误 → 柔顺中心跑到尖端另一侧 → RCC 变"越顺越歪"，插入必卡死。
**3 课**：① 污染质心 c 和零偏，症状是某些姿态残差系统性偏大（非随机）；② 绕该平面法向的旋转分量不可观（所有点绕它转结果不变）。
**4 课**：① 零偏 b 是传感器固有坐标系下的常量，转到 {H} 后它随姿态变化，就不再是常量、没法减；② 差在①：滤波后群延迟占控制周期比例大，迫使加大 D、降低接触速度。
**5 课**：① 稳态力不变（稳态由 D、F_ref 定），瞬态更敏捷也更容易被噪声激励；② 场景：传感器故障输出常值大力 → 偏移无限积分 → 飞车。限幅+安全层双保险。
**6 课**：① 代码用 `max(r, 1e-4)` 下限截断，起步几帧角速度饱和但半径极小，扫过区域不受影响；② 判据不满足（力未降），继续螺旋——毛刺被正确忽略，代价是多花一点搜索时间；③ 横向力矩方向就是卡阻接触点的法平面方向，沿它摆动最直接改变接触状态；随机方向平均要多摆一倍。
**7 课**：① 挪动呈现位姿本身不毁任何标定（它只是个目标点），但会使可达性检查(#15)和治具相关调参失配；挪动**基座**才毁双臂标定；② 一致性检查放行的最坏偏差(5mm)仍在搜孔捕获范围(6mm)内——即使校验擦边通过，下游仍能兜住。
**8 课**：① 真实误差来源就是 UR10 端（夹持重复性、标定漂移），在源头注入才检验整条链路；直接扰动 {H} 测不到坐标链的鲁棒性；② 误检成功污染统计且可能带故障继续下一循环（如插头没插到位就松爪）；压制手段：双条件判据、VERIFY 回拉验证、评测时人工抽查。
